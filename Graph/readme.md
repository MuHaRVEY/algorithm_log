# 3월 30일 학습 정리: 최단 경로

## 1) 최단 경로 문제 분류

최단 경로 문제는 보통 아래 기준으로 나뉜다.

- 단일 출발점 최단 경로 (Single Source Shortest Path)
- 모든 정점 쌍 최단 경로 (All Pairs Shortest Path)
- 가중치가 없는 그래프(또는 모든 간선 가중치가 동일)
- 음수 가중치 포함 여부

어떤 알고리즘을 고를지는 결국 아래 질문으로 정리된다.

- 간선 가중치가 모두 같은가?
- 음수 가중치가 있는가?
- 시작점이 하나인가, 모든 쌍인가?

---

## 2) BFS (비가중치 그래프)

### 핵심 전제

BFS는 간선 가중치가 없거나(비가중치), 모든 간선 가중치가 동일할 때 사용한다.

### 동작 원리

- FIFO(선입선출) 큐를 사용한다.
- 레벨 단위로 탐색한다.
- 먼저 방문한 경로가 곧 최단 경로가 된다.

즉, 어떤 정점에 처음 도달했을 때의 간선 수가 시작점에서 해당 정점까지의 최단 거리다.
간선의 개수가 곧 경로의 길이를 의미한다. 비가중치를 전제로 하므로
간선 1개로 시작... 레벨 단위의 탐색
-> 즉 처음 도착했을 때, 해당 특정 노드까지의 경로가 무조건 최단 경로임을 보장한다.


### 시간 복잡도

- $O(|V| + |E|)$
- 각 정점/간선을 최대 1번씩만 처리한다.

## 3) 다익스트라 Dijkstra (음수 없는 가중치 그래프)

### 핵심 아이디어

다익스트라는 BFS를 가중치 그래프로 확장한 형태로 볼 수 있다.

- 현재까지 최단 거리가 가장 작은 정점을 우선 확정한다.
- 이를 반복하면서 최단 거리 확정 영역을 넓혀 간다.
- 일반적으로 우선순위 큐(최소 힙)를 사용한다.

### 왜 음수 간선이 안 되는가?

다익스트라는 한 번 확정한 최단 거리가 이후에 더 줄어들지 않는다는 성질을 사용한다.
하지만 음수 간선이 있으면 나중에 돌아가는 경로가 더 짧아질 수 있어 이 가정이 깨진다.

### 시간 복잡도 (우선순위 큐 사용)
- 이진 힙을 구성하는데 O(|V|)
- V개의 정점이 힙을 빠져나오며 Heapify할때마다 O(log |V|) -> O(V log V)
- 간선 E에 대해서 거리 갱신 작업이 일어날 수 있음. 힙 내의 키값 조정도 포함하므로 O(E log V)
- 따라서 O((E+V) log V) , E가 좀 더 지배적이므로 E log V로 표현하기도 함.

<정리>
- 정점 추출: $O(|V| \log |V|)$
- 간선 완화(relaxation): $O(|E| \log |V|)$
- 전체: $O((|V| + |E|) \log |V|)$

희소 그래프에서는 보통 $O(|E| \log |V|)$로 많이 표현한다.

## 4) 벨만-포드 Bellman-Ford (음수 가중치 가능)

음수 가중치가 있다면 벨만-포드를 고려한다.

### 핵심 아이디어

- 최단 경로는(음수 사이클이 없다면) 최대 $|V|-1$개의 간선으로 구성된다.
- 모든 간선 $(u, v)$에 대해 완화 작업을 $|V|-1$번 반복한다.
- 한 번 더 완화가 가능하면 음수 사이클이 존재한다.

### 시간 복잡도

- $O(|V||E|)$

---

## 5) 빠른 선택 가이드

- 비가중치(또는 동일 가중치): BFS
- 가중치 존재 + 음수 없음: 다익스트라
- 음수 가중치 존재: 벨만-포드
